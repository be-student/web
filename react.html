개념

  1. render 함수 : 컴포넌트가 어떻게 생겼는지 정의 단 html 을 반환하지 않고, 뷰가 생긴 모양을 저장한 객체를 리턴
  2. 컴포넌트 : 재사용이 가능한 독립된 모듈
  3. 마크업 : tag 와 유사한 뜻
  4. Dom : 문서 객체 모델 XML이나 HTML에 접근하기 위한 인터페이스. 문서내 요소 정의, 접근 방법 제공 XML이나 HTML로 작성됨 동적 UI에 최적화되어있지 않음.(Html이 정적, html+js 동적) DOM 자체는 빠르지만, DOM이 변화하면서 웹 브라우저가 CSS연산, 레이아웃 등등 하면서 길어 짐 하지만, 리액트는 Virtual DOM 이라는 것으로 DOM 업데이트 추상화 함으로 효율적 진행
  5. 조화과정 : 화면 업데이트 과정 render 함수를 호출해서 반환된 결과를 바로 DOM 에 넣지는 않고, 기존 컴포넌트 정보와 비교해서 최소한의 연산으로 처리함
  6. XML : 데이터 전송을 목적으로 디자인 된 언어 html과 차이는 html은 이미 정해진 태그, xml은 태그 정하기 html은 데이터와 표현 동시, xml은 데이터만
  7. Virtual DOM: 실제 DOM 대신 js로 추상화된 객체로 접근. 가벼움 데이터 업데이트시 전체 UI virtual dom에 리렌더링 현재 내용 비교, 바뀐 부분만 DOM 에 반영 DOM 에서 지속적으로 변화하는 데이터가있을 때 효율적임 업데이트 과정의 간결성이 특징
  8. 리액트 : 오직 뷰만 담당하는 특징이 있음. 다른 언어는 백엔드도 터치하는 느낌 라이브러리이기 때문에 기능 구현할게 많음 다양한 프레임 워크, 라이브러리와 혼용 가능
  9. 라이브러리 : 소프트웨어 개발시 컴퓨터 프로그램이 사용하는 비휘발성 자원의 모임. 공통 요소의 기능을 모듈화 한 것
  10. 프레임 워크 : 프로그램의 전체 골격, 뼈대
